Lab Programs Cheat Sheet

Key concepts, pragmas, and explanations

Parallel Merge Sort

Uses #pragma omp parallel sections and #pragma omp section to sort halves simultaneously. Ideal for divide-and-conquer algorithms like merge sort.

Two threads can sort left and right parts at the same time, reducing overall time.

Static Loop Scheduling

Uses #pragma omp for schedule(static,2) to divide loop iterations into fixed chunks of two.

Threads process iterations in a predictable order, which helps balance simple repetitive tasks.

Task-based Fibonacci

Uses #pragma omp task and #pragma omp taskwait to create and synchronize recursive tasks.

Allows different calls to compute in parallel; tasks merge results after finishing.

Parallel Prime Finder

Uses #pragma omp parallel for with reduction(+:count) to safely sum found primes.

Multiple threads check numbers for primality; reduction avoids race conditions.

MPI Send/Recv

Uses MPI_Send and MPI_Recv to exchange data between two processes.

Simple point-to-point communication helps illustrate basic MPI concepts.

MPI Deadlock & Avoidance

Demonstrates how calling send and receive in the same order on both processes can cause deadlock.

Reordering the calls prevents processes from waiting indefinitely for each other.

MPI Broadcast

Uses MPI_Bcast to send data from one process to all others in the communicator.

Efficient when the same data is needed by every process.

MPI Scatter/Gather

MPI_Scatter distributes chunks to each process; MPI_Gather collects results back.

Useful for dividing work and then combining final results.

MPI Reduce

Uses MPI_Reduce and MPI_Allreduce to combine values like sum, min, or max across processes.

Common when calculating totals, averages, or other aggregated data.
